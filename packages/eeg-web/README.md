# @elata-biosciences/eeg-web

Web wrapper for the Elata EEG WASM bindings.

## What This Package Is

`@elata-biosciences/eeg-web` is intentionally a thin wrapper:

- `dist/index.js` and `dist/index.d.ts` provide stable init helpers, wrapper utilities, and compatibility re-exports
- the full EEG API surface is still generated by `wasm-bindgen`, but app code should prefer the package helpers when they exist
- generated WASM assets are shipped in `wasm/`

**This package also includes `RppgPipeline`** for heart-rate estimation from
optical/PPG camera data — despite the name, it is not EEG-only. If you want
low-level rPPG sample ingestion without the managed-session overhead of
`rppg-web`, use `createRppgPipeline()` from this package.

This package does not implement Bluetooth device connection logic by itself.

## When To Use It

**Abstraction level: raw primitives.** This package gives you WASM classes and
functions you call directly — you own the orchestration. It does not manage
sessions, lifecycles, or state machines for you. If you want a managed
camera-capture session with built-in diagnostics, use `rppg-web` instead.

Use `@elata-biosciences/eeg-web` when you want:

- browser-side EEG WASM APIs
- Elata signal processing and model functions in JavaScript or TypeScript
- shared types used by higher-level browser integrations
- low-level rPPG pipeline access without managed-session overhead

Use `@elata-biosciences/eeg-web-ble` in addition if you also need browser BLE
device transport.

## Install

```bash
pnpm add @elata-biosciences/eeg-web
npm install @elata-biosciences/eeg-web
```

## Requirements

- Node.js `>= 20` for server-side usage and local repo tooling
- modern browser with WebAssembly support for in-browser usage

## Usage

```ts
import { initEegWasm, band_powers } from "@elata-biosciences/eeg-web";

await initEegWasm();
const powers = band_powers(eegData, 256);
console.log(powers.alpha);
```

## Recommended Vs Advanced

Recommended:

- For browser rPPG app integration, start with `createRppgSession()` from `@elata-biosciences/rppg-web`.
- If you need low-level rPPG sample ingestion from this package, use `initEegWasm()` plus `createRppgPipeline()`.

Advanced:

- The generated `wasm-bindgen` exports are still re-exported for compatibility and SDK debugging, but they are easier to misuse.
- Avoid instantiating generated wrappers directly unless you are intentionally debugging the SDK itself or need a surface the stable helpers do not expose yet.

If you need the rPPG WASM pipeline from this package, prefer the stable helper
instead of instantiating generated wrappers directly:

```ts
import { initEegWasm, createRppgPipeline } from "@elata-biosciences/eeg-web";

await initEegWasm();
const pipeline = createRppgPipeline(30, 10);
pipeline.push_sample(1000, 0.42);
console.log(pipeline.get_metrics());
```

## Key Exports

- `initEegWasm` and `initEegWasmSync`
- `createRppgPipeline`
- `band_powers`
- `WasmAlphaBumpDetector`
- `WasmAlphaPeakModel`
- `WasmCalmnessModel`
- `AthenaWasmDecoder`
- generated `wasm-bindgen` exports for advanced or compatibility use

## Device Support

Current supported EEG headband integrations in this repo include:

- Muse 2 and Muse S classic BLE devices
- Muse S Athena protocol v2 devices
- the synthetic Muse-compatible BLE bridge for testing

## Build And Dev Notes

From the repo root:

```bash
./run.sh build eeg
```

This builds `eeg-wasm`, runs `wasm-bindgen`, syncs generated files into
`packages/eeg-web/wasm`, and builds the TypeScript wrapper.

For local iteration against an existing app, use:

```bash
./run.sh sync-to ../my-app
```

`sync-to` is a local repo-development helper for `eeg-web`. It is not the
recommended path for starting a new app.

## Package Layout

- `src/*.ts`: source edited in this repo
- `dist/*.js`: emitted runtime files for consumers
- `dist/*.d.ts`: emitted type declarations
- `wasm/*`: generated wasm-bindgen artifacts re-exported by this package

## Troubleshooting

- If `initEegWasm()` fails in a browser app, verify your bundler is serving the packaged `wasm/` assets.
- If you see a deprecated init warning, keep calling `initEegWasm()` or `initEegWasmSync()` rather than passing raw input into generated init exports directly. The wrapper now normalizes the newer object shape for you.
- If the pipeline instance has no `push_sample` or `get_metrics` API, switch to `createRppgPipeline()` instead of constructing generated classes directly.
- If you hit `wasmrppgpipeline_new` errors, make sure you called `await initEegWasm()` before creating the pipeline and avoid bypassing the wrapper helpers.
- If you are testing local repo changes in another app, prefer `./run.sh sync-to ../my-app` instead of manually copying build outputs.
- If you need device connection, this package alone is not enough; add `@elata-biosciences/eeg-web-ble` or another transport layer.
